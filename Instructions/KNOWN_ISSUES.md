# 알려진 이슈와 확인 결과

## 해결된 핵심 이슈

- 트랙 벽 관통: `VehicleController`가 매 physics step마다 벽 방향 속도를 덮어쓰던 구조를 wall-slide·분리 힘·속도 클램프로 바꿨다. 실제 플레이테스트에서 직진·빗각·급커브·부스트 상황을 통과했다.
- 투명벽 끼임: `TrackGeometry.SafeLateralOffset`으로 급커브의 offset 곡선 자기교차를 막아 트랙 표면과 벽의 꼬임을 제거했다.
- 온라인 폭탄 기절·아이템 캠핑: 오너 RPC 지속시간과 재스폰 반경 검사를 보완해 실제 2인 테스트를 통과했다.

## 현재 제약

- Relay/Lobby는 패키지·자동 테스트까지 완료했다. Unity Cloud 방 코드의 2인 라이브 재검증이 남아 있다.
- NetworkRace 씬 파일에 사용자의 미커밋 변경이 있다. 이 파일을 재생성하거나 커밋에 섞지 않는다.
- 아이템 최종형의 상세·스프라이트와 로컬 레이스 HUD는 정식 슬롯/상세 UI로 교체됐다. 작은 Free Aspect 미리보기에서는 고해상도 Canvas 레이아웃이 축소되어 보일 수 있으므로, 최종 해상도별 가독성 조정은 UI 후속 작업으로 남겨 둔다.
- 스피드전과 호스트 방 설정 UI가 구현됐다. PlayMode 51건은 50 통과·실패 0이며, 스크린샷 환경 확인 1건만 inconclusive다.
- 런타임 스테이지 해저드는 `Assets/Prefabs/Stage`의 번들 모델 프리팹을 사용한다. 원본 모델에 전용 텍스처가 없는 경우에는 무채색/무드 tint를 적용하므로, 최종 아트 방향이 확정되면 머티리얼 교체를 별도 검토한다.
- 아이템 파생형은 10% 승급 확률을 유지하면서 계열별 가중치로 추첨한다. 원자폭탄은 전체 스폰의 약 0.4%로 낮췄다.
- 비키니 시티 IP 리스킨 여부는 의도적으로 미결정이다. 이번 작업에서는 어떤 리스킨도 수행하지 않았다.

## 32차 실제 화면 확인

Unity Play 모드에서 다음을 확인했다.

- 비키니 시티: 차량, 트랙, 바위 배경 소품, 산소 HUD, 방패 월드 스프라이트.
- 아프리카TV: 차량·트랙, 멘탈 HUD.
- 네더 요새: 차량·트랙, 체온 HUD가 약 +1/초로 상승, 월드 아이템 스프라이트.
- NetworkBootstrap: 아이템전 방 설정 패널(모드·바퀴·승리 조건)을 Play 모드로 확인했다. 실제 Relay 방 생성 버튼은 인터넷 제한에 따라 누르지 않았다.
- 비키니 시티: 생성된 스테이지 아트 프리팹 라이브러리를 포함한 상태로 레이스 진입을 다시 확인했다.

발견 사항: Free Aspect의 작은 Game View에서는 1920 기준 UI가 축소되어 보인다. 실제 해상도별 레이아웃 미세 조정은 `UI 실제 구현` 후속 단계에서 처리한다.
