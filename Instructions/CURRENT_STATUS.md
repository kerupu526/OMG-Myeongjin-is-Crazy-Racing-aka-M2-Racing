## 레이스 규칙

- `VictoryCondition.SimpleFinish`: 먼저 목표 바퀴를 완주한 레이서가 승리한다.
- `VictoryCondition.StarBet`: 완주자 전체의 별점을 비교하고, 동점이면 완주시간으로 판정한다. 아무도 완주하지 못하면 무승부, 별점·시간까지 같으면 무승부다.
- 산소 부족·화상은 2인 경기에서는 상대 승리, 상대가 없는 경우에는 사유가 있는 무승부다. 원자폭탄은 즉시 무승부로 끝낸다.

## 32차 작업 — UI 실제 구현

- 로컬 스테이지의 개발용 텍스트 HUD를 정식 인게임 레이아웃으로 교체했다. 상단에는 바퀴·대전 상태·남은 시간을, 좌하단에는 주/보조 아이템 슬롯과 조작 힌트를, 하단에는 공통 스테이지 게이지 바를 배치한다.
- 아이템 슬롯은 `ItemSpriteLibrary`의 안전한 스프라이트와 tint를 그대로 사용하고, 선택 아이템의 `ItemDefinition.description`을 상세 패널에 표시한다. 빈 슬롯일 때도 다음 픽업에서 무엇이 표시되는지 안내한다.
- 비키니시티·아프리카TV·네더요새의 이전 개별 게이지 라벨과 차량 충돌 텔레메트리는 정식 HUD가 있을 때 숨긴다. 브리핑·카운트다운·결과 패널은 레이스 HUD와 분리되어 표시된다.
- `RaceHudPresentationTests`가 컴팩트 카드 크기, 공통 게이지 설치, 아이템 획득 뒤 실제 스프라이트·상세 설명 표시를 검증한다. 수동 Unity Play 모드에서도 브리핑 버튼 문구와 레이스 HUD의 비중첩 배치를 확인했다.

## 32차 작업 — 스피드전과 방 설정

- 방 생성 전 `RoomSettingsUI`에서 아이템전/스피드전을 전환하고, 아이템전에서는 1·3·5바퀴와 단순 완주/별점 내기를 순환 선택한다.
- 스피드전은 규칙상 5바퀴·단순 완주·최고 100km/h로 고정된다. 랜덤 트랙 아이템은 비활성화하고, 모든 참가자에게 기본 휘발유를 레이스 시작 시와 이후 5초 간격으로 슬롯을 거치지 않고 자동 분사한다.
- 호스트가 고른 모드·목표 바퀴·속도 제한은 `NetworkRaceManager`의 NetworkVariable로 클라이언트에도 동기화된다. 방 코드와 방 설정은 기존 상태 UI와 같은 고정 픽셀 가독성으로 키웠고, 로컬 HUD·공통 게이지도 같은 기준으로 확대했다.

## 32차 작업 — 정식 스테이지 아트 프리팹

- `Assets/Prefabs/Stage`에 번들 Kenney 모델 기반의 지형 바위·방송 타워·용암 바위·냉각 아치 프리팹을 생성하고, `StageArtPrefabLibrary`를 통해 런타임 스테이지 조립 코드가 참조한다.
- 비키니 시티의 지형 해저드와 네더 요새의 용암 테두리는 이제 충돌 프록시와 분리된 모델 프리팹을 렌더링한다. 아프리카TV 방송사고 구역에는 방송 타워, 네더의 냉각 구역에는 요새 아치를 추가했다.
- `BuildCheck.BuildStageArtPrefabs` 및 프리팹 스모크 테스트를 통과했고, PlayMode는 52건 중 51 통과·실패 0·기존 스크린샷 1건 inconclusive였다. 비키니 시티 Play 모드에서도 레이스 진입과 정식 HUD가 함께 정상 동작함을 확인했다.

## 32차 작업 — 아이템 파생형 가중치

- 계열 선택은 기존처럼 균등하고, 10%의 파생형 승급 확률도 유지한다. 파생형 안에서는 배열 순서에 의존하지 않는 명시적 합계 100 가중치로 선택한다.
- 가속: 슈퍼 휘발유 60, 해피버스데이투유 28, 재석 유 12. 공격: C4 23, 다이너마이트 28, 막대형 수류탄 22, 원자폭탄 4, 러브레터 23. 방어: 가시 방패 72, 황금방패 28.
- 원자폭탄은 공격 파생형의 4%이므로 전체 아이템 스폰에서는 약 0.4%가 된다. `ItemSpawnWeightTests`가 가중치와 각 경계 선택을 고정 검증한다.

## 다음 작업 — UI 실제 구현 후속 1차

- `NetworkBootstrap`의 런타임 메뉴는 1280×720 기준 `Scale With Screen Size`로 통일했다. 작은 Free Aspect 미리보기에서도 방 설정 패널과 방 만들기/참가 조작부가 함께 축소된다.
- `GameManager.LastRaceResults`가 종료 시 모든 참가자의 완주·시간·별점 결과를 보존한다. 결과 모달은 모드·승리 조건·최종 순위·완주시간·별점을 실제 데이터로 표시한다.
- 브리핑과 결과를 금색 테두리의 중앙 카드로 정리했고, 시작 버튼은 브리핑 카드 내부에 배치한다. PlayMode 54건은 53 통과·실패 0·기존 스크린샷 1건 inconclusive였다.

## 32차 작업 — UI 실제 구현 후속 2차

- `NetworkMenuUI`가 HTML의 공통 색·폰트·카드 톤을 적용한 메인, 방 만들기, 방 참가, 대기 로비, 아바타, 환경설정 화면을 런타임에 구성한다. 기존 Relay 방 만들기·참가 컨트롤은 이 화면 안으로 재배치되어 기존 서비스 호출을 유지한다.
- 아바타는 현재 레이서 이름과 3색 선택을 `PlayerPrefs`에 저장하고 메인·로비·결과의 로컬 레이서 표기에 반영한다. 상대 프로필 동기화와 세부 얼굴/장식 선택은 다음 작업이다.
- 환경설정은 마스터 볼륨과 전체 화면 값을 저장한다. 볼륨은 즉시 `AudioListener`에 적용하고, 전체 화면은 독립 실행 빌드에서 적용한다. BGM/SFX·그래픽·언어는 아직 구현하지 않았다.
- 결과 카드의 장문 2인·5바퀴 요약은 760×620 기준 카드 안에 들어가도록 검증했고, 레이스 HUD는 시야 확보를 위해 기존 정식 크기에서 10% 축소했다.
- 이번 후속 UI 직접 PlayMode 검증은 프로필 2건, 설정 1건, 메뉴 2건, 결과 3건이 통과했다. 이어서 전체 PlayMode 66건을 다시 실행해 66건 모두 통과했다.
- Unity 6에서 `InputTestFixture`가 활성 UI 입력 상태를 바꾸며 생기던 `statePtr` 예외는, 테스트별 가상 키보드만 생성·정리하도록 격리 방식을 바꿔 해결했다. 차량 런타임 입력 액션도 차량 파괴 시 해제한다.

## 32차 작업 — 온라인 레이스 HUD 정식화

- `NetworkRaceHUD`의 생성용 정보 라벨과 중앙 배너는 런타임에서 숨기고, 로컬 `RaceHUD`와 같은 1280×720·90% 스케일 기준의 카드형 레이아웃을 구성한다. 상단에는 내 바퀴·남은 시간·대전 상태를, 좌하단에는 주/보조 슬롯과 실제 아이템 스프라이트를, 하단에는 선택 아이템 상세와 조작·스피드전 자동 휘발유 안내를 배치한다.
- 브리핑·카운트다운과 레이스 종료는 별도의 중앙 카드로 표시한다. 종료 카드는 현재 동기화된 승패·무승부 사유·양쪽 바퀴를 표시하며, 상대 이름·아바타·순위·완주 시간·별점은 다음 온라인 결과 동기화 작업에서 연결한다.
- `NetworkRaceHudPresentationTests`가 기존 임시 라벨 비활성화, 공통 Canvas 스케일, 카드 크기, 아이템 아이콘, 카운트다운·결과 카드 생성을 검증한다. 추가 후 Unity 전체 PlayMode 67건이 모두 통과했다.
