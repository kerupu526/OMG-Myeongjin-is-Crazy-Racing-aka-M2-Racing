# 다음 작업 메모

기준일: 2026-07-13

## 다음 작업 — UI 실제 구현

기준 디자인은 사용자가 제공한 `M2 Racing UI.html`이다. 현재의 코드 기반 HUD·방 설정은 기능 확인용 임시 표현이므로, 다음 작업에서는 HTML의 화면 구성·색·타이포그래피·여백·버튼 상태를 Unity UI로 실제 반영한다. 아래 여섯 화면은 하나라도 생략하지 않는다.

1. **메인** — 시작 진입, 핵심 메뉴, 배경과 CTA를 HTML 메인 화면과 동일하게 구현한다.
2. **로비** — 방 만들기/참가, 크게 표시되는 방 코드, 방 설정과 대기 상태를 HTML 로비와 동일하게 구현하고 현재 Relay 기능에 연결한다.
3. **인게임** — 바퀴·시간·상태·스테이지 게이지·아이템/스피드전 자동 보급 정보를 HTML 인게임 화면과 동일한 계층과 크기로 구현한다.
4. **결과** — 순위·완주시간·별점·승리 조건을 실제 `GameManager` 결과 데이터로 채우면서 HTML 결과 화면과 동일하게 구현한다.
5. **아바타** — 아바타 선택/표시 화면과 선택 상태를 HTML 아바타 화면과 동일하게 구현한다.
6. **설정** — 조작·사운드·표시 등 설정 화면과 상태 표현을 HTML 설정 화면과 동일하게 구현한다.

공통 기준:

- 여섯 화면은 개별 임시 패널이 아니라 공통 UI 스타일/컴포넌트로 구성하고, 화면 전환과 기존 게임 흐름에 실제 연결한다.
- 현재 확대한 방 코드와 HUD는 가독성 보완본이다. HTML 구현 단계에서 디자인 원본의 크기와 배치로 교체하되, 방 코드는 멀리서도 읽히는 우선순위를 유지한다.
- 각 화면 묶음은 Unity Play 모드 실제 화면 확인과 PlayMode 회귀를 거친 뒤 기능 단위별로 커밋한다.
